# Phase 7.5: Guardar y Reordenar Rutas — Design Spec

**Date:** 2026-04-16
**Branch:** `feature/phase-7.5-saved-routes`
**Status:** Approved

---

## Overview

Extend the existing route feature (Phase 7) with persistence and editing:
- Save calculated routes to Supabase with a user-chosen name
- List saved routes per user (private via RLS)
- Edit stop order via drag-and-drop and recalculate

## Data Layer

### Supabase migration

File: `supabase/migrations/20260416000001_create_saved_routes_table.sql`

```sql
CREATE TABLE saved_routes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  origin JSONB NOT NULL,
  stops JSONB NOT NULL,
  distance_meters INT NOT NULL,
  duration_seconds INT NOT NULL,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);

ALTER TABLE saved_routes ENABLE ROW LEVEL SECURITY;

CREATE POLICY "User can view own saved routes"
  ON saved_routes FOR SELECT USING (user_id = auth.uid());
CREATE POLICY "User can insert own saved routes"
  ON saved_routes FOR INSERT WITH CHECK (user_id = auth.uid());
CREATE POLICY "User can update own saved routes"
  ON saved_routes FOR UPDATE USING (user_id = auth.uid());
CREATE POLICY "User can delete own saved routes"
  ON saved_routes FOR DELETE USING (user_id = auth.uid());

CREATE INDEX saved_routes_user_id_idx ON saved_routes(user_id);
```

### JSONB schema

**origin:**
```json
{ "lat": 9.934, "lng": -84.087, "displayName": "Mi ubicación", "type": "currentLocation" }
```

**stops array:**
```json
[{ "id": "uuid", "promotionId": "uuid|null", "name": "Supermercado", "lat": 9.93, "lng": -84.08, "order": 1 }]
```

Only store what's needed — no optional LocationEntity fields (accuracy, altitude, etc.).

### Domain entities

**`lib/features/route/domain/entities/saved_route_entity.dart`**

```dart
class SavedRouteEntity extends Equatable {
  final String id;
  final String userId;
  final String name;
  final RouteOriginEntity origin;
  final List<RouteStopEntity> stops;
  final int distanceMeters;
  final int durationSeconds;
  final DateTime createdAt;
  final DateTime updatedAt;
}
```

Uses existing `RouteOriginEntity` and `RouteStopEntity`.

### Domain repository (abstract)

**`lib/features/route/domain/repositories/saved_routes_repository.dart`**

```dart
abstract class SavedRoutesRepository {
  Future<Either<Failure, List<SavedRouteEntity>>> getSavedRoutes();
  Future<Either<Failure, SavedRouteEntity>> createSavedRoute(SavedRouteEntity route);
  Future<Either<Failure, SavedRouteEntity>> updateSavedRoute(SavedRouteEntity route);
  Future<Either<Failure, void>> deleteSavedRoute(String routeId);
}
```

### Data model

**`lib/features/route/data/models/saved_route_model.dart`**

Extends `SavedRouteEntity`. Implements `fromSupabase(Map)` and `toSupabase()` that manually serialize/deserialize JSONB. Reconstructs `LocationEntity` with only lat/lng and `DateTime.now()` as timestamp placeholder.

### Data source

**`lib/features/route/data/datasources/saved_routes_remote_data_source.dart`**

```dart
abstract class SavedRoutesRemoteDataSource {
  Future<List<SavedRouteModel>> getSavedRoutes();
  Future<SavedRouteModel> createSavedRoute(SavedRouteModel route);
  Future<SavedRouteModel> updateSavedRoute(SavedRouteModel route);
  Future<void> deleteSavedRoute(String routeId);
}
```

Implementation uses Supabase client:
- `getSavedRoutes` → `.from('saved_routes').select().order('created_at', ascending: false)`
- `createSavedRoute` → `.insert(data).select().single()`
- `updateSavedRoute` → `.update(data).eq('id', id).select().single()`
- `deleteSavedRoute` → `.delete().eq('id', id)`

Throws `ServerException` on error. Repository wraps in `StorageFailure`.

### Repository impl

**`lib/features/route/data/repositories/saved_routes_repository_impl.dart`**

Standard wrapper: try/catch datasource calls, fold exceptions into `StorageFailure`.

## BLoC

**`lib/features/route/presentation/bloc/saved_routes_bloc.dart`**

### Events

```dart
class SavedRoutesLoadRequested extends SavedRoutesEvent
class SavedRouteCreateRequested extends SavedRoutesEvent {
  final SavedRouteEntity route;
  final String name;
}
class SavedRouteUpdateRequested extends SavedRoutesEvent {
  final SavedRouteEntity route; // contains reordered stops
}
class SavedRouteDeleteRequested extends SavedRoutesEvent {
  final String routeId;
}
```

### States

```dart
class SavedRoutesInitial extends SavedRoutesState
class SavedRoutesLoading extends SavedRoutesState
class SavedRoutesLoaded extends SavedRoutesState {
  final List<SavedRouteEntity> routes;
}
class SavedRouteOperationInProgress extends SavedRoutesState
class SavedRoutesError extends SavedRoutesState {
  final String message;
}
```

No separate `SavedRouteLoaded` state — `RouteEditorPage` receives the entity via navigation `extra`.

### Handler logic

Each handler: emit `SavedRouteOperationInProgress` (or `SavedRoutesLoading` for load) → call repo → on success re-emit `SavedRoutesLoaded` with fresh list → on error emit `SavedRoutesError`.

    ..add(const SavedRoutesLoadRequested()),

## UI Pages

### SavedRoutesPage

- AppBar: "Mis rutas guardadas"
- BlocBuilder on `SavedRoutesBloc`
- Empty state: centered text + `ElevatedButton` "Planificar ruta" → push routePlanner
- List: `RefreshIndicator` wrapping `ListView.builder`
  - Each item: `Dismissible` (swipe left) → `AlertDialog` confirm → `SavedRouteDeleteRequested`
  - Card content: name, distance (km), duration (min), `createdAt` formatted
  - "Editar" button → `context.push(AppRouter.routeEditor, extra: route)`

### RouteEditorPage

- Receives `SavedRouteEntity` via `state.extra`
- AppBar: "Editar: {route.name}"
- Origin display: non-interactive `ListTile`
- `ReorderableListView` (Flutter native) for stops
  - Each item: order number, name, `ReorderableDragStartListener` (six-dots icon)
  - On reorder: `setState` updating local list, reassign `order` 1..N
- Bottom buttons:
  - "Guardar orden" → `SavedRoutesBloc.add(SavedRouteUpdateRequested(route: updatedRoute))` → snackbar "Ruta actualizada"
  - "Recalcular" → dispatch `RouteCalculationRequested` via `RoutePlannerBloc` with stops in new order → push routeResult

### RouteResultPage (minimal change)

After `ExportRouteButtons`, add `TextButton.icon` "Guardar ruta":
- Tap → `showDialog` with `TextField` for name (required, validated)
- Confirm → `SavedRoutesBloc.add(SavedRouteCreateRequested(...))` → snackbar "Guardada como '{name}'"

## Decisions NOT taken

- No polylines stored in DB — recalculated on-demand
- No gamification integration (Phase 8+)
- No `reorderable_grid` package — Flutter native `ReorderableListView` sufficient
- No undo on swipe-delete (out of scope for 7.5)
